1. Escalabilidad

Concepto: La escalabilidad es la capacidad que tiene un sistema para soportar una mayor cantidad de usuarios o solicitudes sin perder rendimiento. Es importante porque permite que una aplicación siga funcionando correctamente cuando aumenta la demanda.

Ejemplo: Durante el período de inscripciones, miles de alumnos ingresan al sistema al mismo tiempo y este sigue funcionando sin volverse lento.

2. HTTP

Concepto: HTTP es un protocolo que permite la comunicación entre un cliente y un servidor a través de internet. Gracias a él podemos acceder a páginas web y aplicaciones online.

Ejemplo: Cuando abrimos una página web, el navegador utiliza HTTP para solicitar información al servidor.

3. SOAP

Concepto: SOAP es un protocolo que permite intercambiar información entre sistemas siguiendo una estructura definida. Se utiliza principalmente en aplicaciones empresariales.

Ejemplo: Un banco puede utilizar SOAP para consultar datos de clientes desde otro sistema.

4. Interoperabilidad

Concepto: La interoperabilidad permite que sistemas desarrollados con tecnologías diferentes puedan compartir información y trabajar juntos sin inconvenientes.

Ejemplo: Un sistema académico desarrollado en Java puede comunicarse con un sistema de pagos desarrollado en otro lenguaje.

5. Estilo Arquitectónico

Concepto: Es la forma en que se organiza un sistema y cómo se relacionan sus componentes. Cada estilo tiene ventajas según las necesidades del proyecto.

Ejemplo: Cliente-Servidor y Microservicios son estilos arquitectónicos diferentes.

6. Escalamiento Fino

Concepto: Consiste en aumentar recursos solamente en la parte del sistema que los necesita, sin modificar el resto de la aplicación.

Ejemplo: Si el módulo de inscripciones recibe muchas solicitudes, se amplía únicamente ese servicio.

7. Tolerancia a Fallos

Concepto: Es la capacidad de un sistema para continuar funcionando aunque alguna de sus partes presente errores o deje de funcionar.

Ejemplo: Si un servidor falla, otro puede seguir atendiendo a los usuarios.

8. Complejidad Operativa

Concepto: Se refiere a la dificultad que implica administrar, supervisar y mantener un sistema en funcionamiento.

Ejemplo: Gestionar cientos de microservicios requiere más trabajo que administrar una sola aplicación.

9. IaaS

Concepto: IaaS es un modelo de computación en la nube que ofrece infraestructura como servidores y almacenamiento a través de internet.

Ejemplo: AWS EC2 permite alquilar servidores virtuales para alojar aplicaciones.

10. PaaS

Concepto: PaaS es un servicio que proporciona una plataforma lista para desarrollar y publicar aplicaciones sin preocuparse por la infraestructura.

Ejemplo: Heroku permite desplegar aplicaciones sin configurar servidores manualmente.

11. SaaS

Concepto: SaaS es un modelo donde el software se utiliza directamente desde internet sin necesidad de instalarlo localmente.

Ejemplo: Gmail y Office 365 son ejemplos de Software como Servicio.

12. HDFS

Concepto: HDFS es un sistema de archivos distribuido que permite almacenar grandes cantidades de información en varios servidores.

Ejemplo: Una universidad puede almacenar datos de miles de alumnos distribuidos en distintos equipos.

13. Wiki Colaborativa

Concepto: Es un sitio donde varias personas pueden crear, modificar y compartir contenido de manera conjunta.

Ejemplo: Varias escuelas pueden colaborar en una misma wiki educativa.

14. Blockchain

Concepto: Blockchain es una tecnología que almacena información en múltiples equipos conectados entre sí para aumentar la seguridad y la transparencia.

Ejemplo: Las transacciones quedan registradas en diferentes computadoras al mismo tiempo.

15. Redes Peer-to-Peer

Concepto: Son redes donde los dispositivos se comunican directamente entre sí sin depender de un servidor central.

Ejemplo: Los usuarios pueden compartir archivos directamente entre sus computadoras.

16. Microservicios

Concepto: Los microservicios son un estilo arquitectónico que divide una aplicación en servicios pequeños e independientes, cada uno encargado de una función específica.

Ejemplo: Un sistema puede tener un microservicio para usuarios, otro para pagos y otro para inscripciones.

17. Heroku

Concepto: Heroku es una plataforma en la nube que permite publicar aplicaciones de forma rápida y sencilla.

Ejemplo: Un desarrollador puede subir una aplicación web y dejarla disponible para los usuarios.

18. AWS EC2

Concepto: AWS EC2 es un servicio de Amazon que proporciona servidores virtuales para alojar aplicaciones y sistemas.

Ejemplo: Una universidad puede ejecutar su sistema de inscripciones utilizando AWS EC2.

19. Apache Kafka

Concepto: Apache Kafka es una herramienta diseñada para transportar grandes volúmenes de información entre distintas aplicaciones de forma rápida y eficiente.

Ejemplo: Cuando un alumno se inscribe, Kafka puede enviar automáticamente esos datos a otros sistemas.

20. Docker

Concepto: Docker es una herramienta que permite empaquetar una aplicación junto con todas sus dependencias para ejecutarla de la misma forma en distintos entornos.

Ejemplo: Una empresa puede desarrollar un microservicio y ejecutarlo en varios servidores sin modificar su configuración.

21. Monolito

Concepto: Un monolito es una aplicación donde todas las funcionalidades se encuentran integradas dentro de un único programa.

Ejemplo: Usuarios, pagos e inscripciones forman parte de una misma aplicación.

22. Servidor

Concepto: Un servidor es una computadora o programa encargado de almacenar información y responder a las solicitudes realizadas por otros dispositivos.

Ejemplo: Cuando abrimos Instagram, los servidores envían las fotos y videos que vemos en la aplicación.

23. Plataforma

Concepto: Una plataforma es un entorno que proporciona herramientas y servicios para desarrollar, administrar o utilizar aplicaciones.

Ejemplo: Heroku es una plataforma que facilita la publicación de aplicaciones web.

24. Sistemas Distribuidos

Concepto: Son sistemas formados por varias computadoras que trabajan juntas para ofrecer un servicio o realizar una tarea.

Ejemplo: Google utiliza miles de servidores distribuidos para responder búsquedas rápidamente.

25. Nodos

Concepto: Los nodos son las computadoras o servidores que forman parte de una red o sistema distribuido.

Ejemplo: Cada servidor que participa en la infraestructura de Google puede considerarse un nodo.
